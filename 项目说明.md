# 校园二手交易平台数据库系统 - 项目说明

## 1. 在线访问地址

- 本地开发地址：`http://127.0.0.1:5000`
- 如需公网演示，可部署到 Render / Railway 等平台。

## 2. 从代码到运行的步骤

1. 安装依赖：`pip install -r requirements.txt`
2. 启动系统：`python app.py`
3. 浏览器访问：`http://127.0.0.1:5000`
4. 如需恢复初始数据：管理员登录后，在首页点击“重建数据库并恢复初始数据”

## 3. 页面与结果说明

- 首页：导航、核心统计、重建数据库入口（仅管理员可见）
- 用户列表页：展示 `user` 表
- 商品列表页：展示 `item` 表，并提供写操作（仅管理员）
  - 新增商品
  - 修改商品价格
  - 删除未售商品
  - 购买商品
- 订单列表页：展示 `orders` 表及订单详情
- 查询展示页：
  - 基本查询
  - 连接查询
  - 聚合分组
  - 视图查询
  - 图表可视化（Chart.js）

## 4. 第八部分：安全性说明

### 4.1 如何防止普通用户删除数据

- 在应用层实现角色控制：
  - `viewer` 仅允许读操作（GET）
  - `admin` 才允许写操作（POST）
- 写操作入口统一受控：新增/修改/删除/购买/重建数据库均要求管理员身份

### 4.2 如何限制用户只能查询数据

- 未登录或普通用户访问时，页面仅显示只读内容
- 写接口在服务端再次做权限校验，避免仅靠前端限制
- 所有 SQL 使用参数化语句，降低 SQL 注入风险

## 5. 第九部分：并发与恢复说明

### 5.1 两个用户同时购买同一商品可能出现的问题

若没有事务和约束保护，可能出现重复下单（同一商品产生多个订单）。

### 5.2 本项目如何解决

- 购买流程使用事务起点：`BEGIN IMMEDIATE`
- 数据库唯一约束：`orders.item_id UNIQUE`
- 触发器校验：
  - 商品不存在时阻止下单
  - 已售商品阻止再次购买
  - 买家阻止购买自己发布的商品
- 下单成功后由触发器自动更新商品状态为已售

### 5.3 系统崩溃后的恢复

- SQLite 事务保证原子性：未提交事务会自动回滚
- 可通过“重建数据库并恢复初始数据”快速回到可演示状态
- 线上环境建议配合定时备份策略

## 6. 演示账号

- 普通用户：`viewer / viewer123`

## 7. 自动化测试

执行命令：`python -m unittest discover -s tests -v`

已覆盖关键场景：
- 普通用户写操作被拒绝
- 管理员写操作成功
- 购买自己发布的商品被阻止（应用层与数据库层）
- 已成交商品禁止删除
- 重建数据库权限校验
